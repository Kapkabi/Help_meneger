Создай desktop-приложение для хранения и управления рецептами на Python
с использованием PySide6 (GUI) и SQLite (хранение данных).

Базовый функционал:
1. Добавление рецепта: название, список ингредиентов (с количеством и
   единицей измерения), шаги приготовления, время готовки, категория
   (завтрак/обед/ужин/десерт/др.), фото (опционально, путь к файлу)
2. Просмотр списка рецептов с превью (название + категория + миниатюра)
3. Просмотр детальной карточки рецепта
4. Редактирование и удаление рецепта
5. Поиск и фильтрация по названию, категории, ингредиенту
6. Сортировка (по названию, времени готовки, дате добавления)

Архитектура:
- Разделяй слои: UI (виджеты) / бизнес-логика / доступ к данным
  (паттерн MVC или MVVM — выбери сам, что лучше ляжет на PySide6)
- База данных: SQLite, доступ через отдельный data-layer модуль
  (не пиши SQL-запросы прямо в коде виджетов)
- Спроектируй схему БД с расширяемостью на будущее: возможность позже
  добавить теги, рейтинг рецептов, импорт/экспорт, multi-user режим,
  историю изменений — заложи это в структуру таблиц и модулей сейчас,
  даже если сами фичи не реализуешь
- Используй миграции схемы БД (например, простую систему версионирования
  схемы), чтобы можно было безопасно добавлять поля в будущем

Технические требования:
- Python 3.11+, PySide6 (НЕ PyQt6 — нужна именно LGPL-версия)
- Структура проекта: разделение на пакеты (ui/, core/, data/, models/)
- requirements.txt с зафиксированными версиями зависимостей
- Базовая обработка ошибок (повреждённая БД, некорректный ввод и т.п.)
- Юнит-тесты для слоя работы с БД и бизнес-логики (pytest)
- README с описанием структуры проекта и инструкцией запуска

Все архитектурные и реализационные решения (структура папок, выбор
паттерна, конкретные виджеты PySide6, схема БД, организация миграций)
принимай самостоятельно, без уточняющих вопросов — ориентируйся на
лучшие практики для desktop-приложений на PySide6 и SQLite.
